# PROJECT COMPASS — Token Hunter

## 1. Identita produktu

**Název:** Token Hunter  
**Pracovní tagline:** Lovec slev pro LLM modely, API a AI tooling.

### Co produkt je
Token Hunter je savings-first produkt, který sleduje:
- ceny LLM/API modelů,
- free tiery,
- trialy a kredity,
- promo akce,
- levnější přístupové cesty ke stejnému modelu,
- bundle/value příležitosti,
- relevantní novinky v AI a vibe codingu jen tehdy, pokud mohou změnit nákladovou nebo praktickou výhodnost.

### Co produkt není
Token Hunter není:
- obecný AI news reader,
- benchmark portál pro akademické srovnávání,
- API gateway,
- App Store pro modely,
- diskuzní fórum,
- katalog všech možných nabídek bez verdiktu.

## 2. Hlavní problém

AI svět je roztříštěný. Uživatel dnes musí ručně zjišťovat:
- kde je stejný model levnější,
- jestli je „free“ opravdu free nebo jen marketingově pojmenovaný trial,
- jaké jsou limity, regionální restrikce a rate limity,
- co je skutečná příležitost a co jen hlučný launch.

Token Hunter tenhle chaos překládá do jedné věty:

> Co je teď pro můj use-case nejvýhodnější koupě?

## 3. Primární cíl

Dodat uživateli **rozhodnutí**, ne další tabulku.

Produkt má umět během pár sekund odpovědět:
- co je dnes nejvýhodnější,
- kde může ušetřit,
- jaká alternativa dává lepší value,
- co stojí za sledování,
- co je hype a co má praktický dopad.

## 4. Primární cílovka

### Core user
- AI power user
- indie hacker
- vibe coder
- builder, co platí více nástrojů a nechce přeplácet
- nerd, co rád sleduje modely, ale nechce pročítat desítky zdrojů denně

### Secondary user
- tech lead / engineering manager
- founder
- solo dev
- člověk, co chce rozumný poměr cena/výkon pro osobní AI stack

## 5. Jobs to be done

Uživatel si chce říct:
- „Chci nejlevnější rozumný model na coding.“
- „Chci free nebo skoro free variantu na experimenty.“
- „Upozorni mě, když se objeví lepší deal.“
- „Chci vědět, kdy je stejný model výhodnější přes jinou route.“
- „Chci vědět, jestli tahle novinka stojí za pozornost.“
- „Nechci číst 40 blogů denně. Chci verdict.“

## 6. Produktová teze

Token Hunter vyhrává tehdy, když není nejširší databází, ale **nejlepším rozhodovačem**.

To znamená:
- méně položek,
- vyšší důvěra,
- jasnější verdict,
- lepší alerty,
- méně šumu,
- rychlejší návratnost otevření aplikace.

## 7. Pevné principy

### P1 — Savings first
Úspora je hlavní osa produktu. News a discovery vrstva je doplněk.

### P2 — Official-first truth
Ceny, limity a podmínky mají primárně vycházet z oficiálních zdrojů.

### P3 — Best buy, ne raw dump
Produkt musí dodat shortlist, důvod, limity, rizika a alternativy.

### P4 — Trust over speed
Raději o něco pozdější, ale ověřený verdict než rychlý nesmysl.

### P5 — No hallucinated offers
Neověřená sleva, promo nebo kredit nesmí do hlavního feedu.

### P6 — Human-readable rules
Každý deal a každý alert musí jít lidsky vysvětlit.

### P7 — Mobile consumption, server intelligence
Crawling, parsing a scoring patří na backend. Mobil je spotřební vrstva.

### P8 — Burnout guardrails
Produkt nesmí sklouznout do nekonečné encyklopedie. Užitek vítězí nad pokrytím.

## 8. Definice pojmů

### Deal
Konkrétní výhodná nabídka: free tier, trial, kredit, promo, sleva, bundle nebo levnější route.

### Best buy
Nejvýhodnější volba pro konkrétní use-case po započtení:
- ceny,
- limitů,
- frikce,
- kvality,
- stability,
- expirace,
- dostupnosti,
- důvěryhodnosti zdroje.

### Game changer
Novinka, která mění pravidla hry alespoň v jedné rovině:
- výrazně levnější přístup,
- výrazně lepší value,
- nový praktický workflow,
- zásadně nižší bariéra vstupu,
- masivní zjednodušení používání.

### Hype
Hlasitá novinka bez jasného praktického posunu v nákladu, kvalitě nebo workflow.

### Verified
Potvrzeno z oficiálního nebo vysoce důvěryhodného zdroje.

### Watch-worthy
Ještě ne game changer, ale stojí za sledování.

## 9. Anti-goals

Produkt se nesmí změnit v:
- obří katalog bez rozhodování,
- další RSS čtečku,
- benchmark laboratoř,
- slepý crawler všeho,
- affiliate odpad,
- produkt závislý na jednom providerovi,
- nespolehlivý alert spam.

## 10. Produktové pilíře

### A. Savings Radar
Hlavní radar úspory: best buys, free/free-ish cesty, levnější route, expirace.

### B. Best Buy Engine
Rozhodovací vrstva podle use-case:
- coding cheap,
- coding best value,
- chat cheap,
- experimentation,
- agent workflows,
- batch summarization,
- local/open-source oriented.

### C. Verified Discovery
Sekundární vrstva pro novinky a releasy, ale jen po ověření relevance.

### D. Alerts & Watchlists
Důvod, proč se uživatel vrací:
- price drop,
- new free tier,
- cheaper route found,
- expiring soon,
- new relevant release,
- game changer candidate.

## 11. Výherní pozice

Token Hunter nevyhraje tím, že bude mít nejvíc dat. Vyhraje tím, že udělá z roztříštěných dat **nejdůvěryhodnější a nejrychlejší rozhodnutí**.

## 12. Závěrečná věta kompasu

> Token Hunter není katalog modelů. Je to rozhodovací motor, který ti řekne, jak za AI utratit co nejmíň a přitom nekoupit blbost.
